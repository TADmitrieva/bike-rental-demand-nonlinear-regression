# bike-rental-demand-nonlinear-regression
Разработка нелинейного ML-пайплайна (KNN, Decision Tree, Optuna) для почасового прогнозирования спроса на аренду велосипедов в условиях черноморского климата для логистической системы BikeSouth. Достигнуто снижение целевой метрики RMSE на 22% по сравнению с линейным Baseline.

## Описание исследования
В рамках данного проекта исследуется процесс предсказания почасового спроса на аренду велосипедов на основе погодных, календарных и сезонных условий для логистической системы компании **BikeSouth**. Работа охватывает полный цикл разработки ML-решения: от первичного и исследовательского анализа данных (EDA) до оптимизации нелинейных моделей и консервации финального пайплайна в промышленном формате. 

Приоритетными задачами являлись выявление сложных нелинейных взаимосвязей и эффектов взаимодействия признаков (таких как сочетание экстремальной температуры, влажности и осадков), с которыми не справлялась базовая линейная модель.

## Цель исследования
Построить модель машинного обучения, которая максимально точно прогнозирует количество арендованных велосипедов за конкретный час, превосходя по качеству базовую линейную регрессию и обеспечивая минимальное значение метрики **RMSE** как на кросс-валидации обучающей выборки, так и на отложенных тестовых данных.

## Стек технологий
* **Язык разработки:** Python 3.10
* **Анализ данных и EDA:** Pandas, NumPy, Matplotlib, Seaborn, Phik
* **Машинное обучение (ML):** Scikit-Learn (Pipelines, ColumnTransformer, KNeighborsRegressor, DecisionTreeRegressor)
* **Оптимизация гиперпараметров:** Optuna (TPE Sampler)
* **Консервация моделей:** Joblib

## Итоги и выбор финальной модели
На основе минимизации главной целевой метрики проекта (**RMSE = 320.03**) в качестве наилучшего промышленного решения выбран сквозной автоматизированный пайплайн модели **Decision Tree Regressor**. 

**Бизнес-эффект от внедрения:**
1. Величина среднеквадратичной ошибки (RMSE) сократилась более чем на **91 единицу** по сравнению с бейзлайном компании, что сводит к минимуму риск возникновения критических промахов логистики.
2. Средняя абсолютная ошибка (MAE) снизилась на **100 велосипедов в час**. Модель позволяет планировать объемы вывода техники на станции проката BikeSouth на 32% точнее.
3. Доля объясненной дисперсии спроса (R²) выросла с 59% до **75%**. При этом модель обладает стопроцентной бизнес-интерпретируемостью (`Feature Importance`), превращая «черный ящик» алгоритмов в прозрачный инструмент управления.

## Архитектура сохраненного решения (`.joblib`)
Экспортированный артефакт `final_dt_bike_demand_pipeline.joblib` представляет собой полностью автономный монолитный конвейер. При подаче «сырых» данных на вход он автоматически выполняет:
1. Приведение наименований полей к стандарту `snake_case`.
2. Конструирование кастомных нелинейных погодных признаков (синергетический индекс комфорта `temperature_humidity_ratio` и триггерный флаг штормовых осадков `is_bad_weather`).
3. Динамическую импутацию скрытых пропусков метеоданных локальными медианами, рассчитанными на этапе обучения (`fit`).
4. One-Hot кодирование категориальных признаков (`Seasons`, `Holiday`, `Functioning Day`).
5. Стандартизацию числовых осей через `StandardScaler` без риска утечки данных.
